# Muti CLI — 기획서 (Product Specification)

> 버전 0.1 / 작성일 2026-05-11 / 상태: 초안

## 1. 배경 / 목적

Windows에서 여러 프로젝트를 동시에 다루는 개발자·운영자는 일상적으로 `cmd.exe`, PowerShell, WSL 창을 4~10개씩 열어 두고 일한다. 이때 다음과 같은 페인포인트가 반복된다.

- 창들이 작업표시줄에 평탄하게 나열되어 **어느 창이 어느 프로젝트인지 식별이 어렵다.**
- 재부팅 또는 실수 종료 시 **작업 디렉터리·열려 있던 콘솔 개수가 모두 사라진다.**
- 프로젝트별로 환경(시작 폴더, 셸 종류)을 매번 수동으로 다시 세팅해야 한다.

Muti CLI는 위 문제를 해결하기 위한 **설치형 데스크톱 멀티 터미널**이다. 하나의 창 안에서 프로젝트 단위로 터미널을 묶고, 종료해도 상태를 자동 복원한다.

## 2. 타깃 사용자 / 사용 시나리오

### 타깃 사용자

- 풀스택 개발자: 프론트/백/DB 콘솔을 동시에 띄워 두고 작업.
- 운영자(SRE): 여러 서버·도구 콘솔을 프로젝트(=고객사/서비스)별로 묶어 사용.
- 학생/스터디: 강의별로 작업 폴더와 콘솔을 분리해 관리.

### 시나리오

1. **시나리오 A — 풀스택 개발자 "민서"**
   "프론트엔드(Vite dev), 백엔드(uvicorn), DB(psql) 세 콘솔을 한 프로젝트로 묶고 싶다." → 프로젝트 `shop-web`에 탭 3개를 만들고 각각 다른 명령을 띄운다. 노트북을 닫았다 열어도 같은 폴더에서 그대로 이어진다.

2. **시나리오 B — 운영자 "재훈"**
   고객사 A/B/C 콘솔을 분리. 사이드바에서 고객사 전환 시 해당 콘솔 묶음만 보이고 다른 고객사 콘솔은 숨겨진다.

3. **시나리오 C — 학생 "예린"**
   파이썬 강의·자료구조 강의·개인 토이프로젝트 세 폴더를 분리해 사용. 한 탭에서 좌우 분할로 코드 실행과 git을 동시에 본다.

## 3. 핵심 가치 제안

- **프로젝트 단위 묶음**: 작업 폴더 + 탭 그룹이 한 단위. 컨텍스트 전환이 단순.
- **자동 저장·복원**: 모든 변경은 디스크에 저장. 강제 종료에도 복원.
- **설치형 단일 앱**: 별도 런타임 설치 없이 NSIS 설치 파일로 실행. 시작 메뉴/바탕화면 단축키.

## 4. 기능 요구사항

### P0 — 1차 릴리스 (이번 사이클)

- 사이드바: 프로젝트 추가/이름변경/폴더 변경/삭제.
- 탭바: 한 프로젝트 내 여러 터미널 탭 (추가/이름변경/닫기/전환).
- 터미널: xterm 기반 PTY 셸. cwd는 프로젝트 폴더에서 시작.
- **셸 선택**: 프로젝트 디폴트 셸을 `cmd / PowerShell / WSL` 중에서 선택. WSL은 미설치 시 자동 비활성화.
- **탭 분할 뷰**: 한 탭 안에서 좌/우 또는 상/하로 분할(최대 2×2 = 4분할).
- 자동 저장: 변경 시 250ms 디바운스로 `state.json`에 저장.
- 자동 복원: 시작 시 마지막 활성 프로젝트·탭·작업 폴더·분할 레이아웃·창 크기/위치 복원.

### P1 — 다음 (스코프 아님)

- 단축키 (Ctrl+T, Ctrl+W, Ctrl+1..9 등).
- 명령 검색 / Quick Switcher.
- 테마 토글 (라이트/다크).
- 프로젝트별 환경변수 편집 UI.
- 프로젝트별 시작 명령 (탭 생성 시 자동 실행).

### P2 — 백로그

- SSH 프로파일.
- 명령 스니펫 라이브러리.
- 클라우드 동기화.

## 5. 비기능 요구사항

- **성능**: 탭 30개를 띄운 상태에서 활성 탭의 입력 지연 50ms 이하, 스크롤 60fps 유지.
- **보안**: Electron `contextIsolation: true`, `nodeIntegration: false`. 모든 IPC는 preload의 화이트리스트 API만 사용. 외부 입력(URL fetch 등) 없음.
- **호환성**: Windows 10 22H2 이상 / Windows 11. (macOS·Linux는 자연 지원되지만 1차 릴리스 검증 대상 아님.)
- **저장 위치**: `%APPDATA%/Muti CLI/state.json`. 평문 JSON. 마이그레이션 키 `schemaVersion`.
- **국제화**: UI 기본 한국어, 영어 fallback. (이번 사이클은 한국어 고정.)

## 6. 범위 외 (Out of Scope)

- 원격 서버 직접 관리(SSH/SFTP).
- 멀티 윈도우 간 상태 동기화 — v1은 단일 윈도우 가정.
- 자동 업데이트 — 수동 인스톨러 재실행으로 처리.

## 7. 성공 지표

- 최초 실행 후 새 프로젝트 생성 → 탭 2개 추가 → 앱 종료 → 재실행 → 동일 상태 확인까지 **3분 이내**에 완료 가능.
- 사용자 1명이 프로젝트 3개·탭 10개를 24시간 사용한 뒤 state 파일 크기 100KB 이하.
- 강제 종료(작업 관리자 kill) 시에도 마지막 저장 시점 기준으로 복원, 데이터 손실 0.

## 8. 용어 정리

- **프로젝트(Project)**: 작업 폴더 + 환경 설정(디폴트 셸 등) + 탭 묶음. 사이드바 1행에 대응.
- **탭(Tab)**: 하나의 화면 영역. 1개의 패널 또는 분할 트리로 구성.
- **패널(Pane)**: 분할의 리프 노드. 1개의 셸 세션을 보유.
- **세션(Session)**: 패널에 연결된 node-pty 프로세스 인스턴스.
- **상태(State)**: `state.json`에 저장되는 직렬화 가능한 사용자 데이터 전체.
