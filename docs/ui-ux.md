# Muti CLI — UI/UX 계획서

> 버전 0.1 / 작성일 2026-05-11 / 상태: 초안

## 1. 디자인 원칙

1. **다크 우선**: 콘솔 앱이므로 기본 다크. 라이트 모드는 P1.
2. **정보 밀도 우선**: 헤더/패딩 최소화. 같은 화면에서 더 많은 정보를 보여준다.
3. **키보드 친화**: 마우스 없이도 핵심 동작(전환·분할·닫기) 가능해야 한다. (구체 단축키는 v2.)
4. **놀라움 금지**: 동작은 VSCode·Windows Terminal의 익숙한 패턴을 따른다.

## 2. 레이아웃 와이어프레임

### 메인 화면

```
┌─────────────────┬──────────────────────────────────────────────────┐
│ Muti CLI    ＋ │  ┌Tab1┐┌Tab2*┐┌Tab3┐     ＋        ⊟▢ ⊟▭        │
│─────────────────│──────────────────────────────────────────────────│
│ ▸ ProjectA  ●  │                                                  │
│   C:\dev\a    │                                                  │
│   [이름] [폴더] │                                                  │
│   [삭제]       │                  xterm pane                      │
│ ▸ ProjectB     │                  (active)                        │
│   C:\dev\b    │                                                  │
│ ▸ ProjectC     │                                                  │
│   D:\srv\c    │                                                  │
│                │                                                  │
│ shell: cmd ▾   │                                                  │
└─────────────────┴──────────────────────────────────────────────────┘
```

기호 의미:
- `●` 활성 프로젝트 표시
- `*` 활성 탭 표시
- `＋` 추가 버튼
- `⊟▢` 좌우 분할 / `⊟▭` 상하 분할

### 분할 상태 (2분할 예시)

```
┌──────────────────────────┬───────────────────────────┐
│  pane A (active)         │  pane B                   │
│                          │                           │
│  > npm run dev           │  > git status             │
│                          │                           │
└──────────────────────────┴───────────────────────────┘
```

### 빈 상태

```
┌───────────────────────────────────────────────────────┐
│                                                       │
│           프로젝트가 없습니다                          │
│           [＋ 새 프로젝트 만들기]                      │
│                                                       │
└───────────────────────────────────────────────────────┘
```

### 셸 종료 상태 (패널 내부)

```
┌───────────────────────────────────────────────────────┐
│  [프로세스 종료됨 — exit code 1]      [▶ 재시작]      │
│                                                       │
│  (마지막 출력이 남아 있음)                            │
└───────────────────────────────────────────────────────┘
```

## 3. 컴포넌트 인벤토리

| 컴포넌트 | 책임 |
|---|---|
| `Sidebar` | 브랜드, 프로젝트 목록 컨테이너, 새 프로젝트 버튼 |
| `ProjectListItem` | 프로젝트 1개 표시. 활성 시 인라인 액션(이름/폴더/삭제) 노출 |
| `ShellDropdown` | 사이드바 하단의 디폴트 셸 선택. 활성 프로젝트에 귀속 |
| `TabBar` | 탭 목록, 새 탭 버튼, 분할 버튼 |
| `TabItem` | 탭 1개. 더블클릭 시 이름 변경, × 로 닫기 |
| `SplitContainer` | 좌우/상하 분할 컨테이너. 가운데에 드래그 가능한 splitter |
| `TerminalPane` | xterm 인스턴스를 가진 리프 패널. 헤더에 패널 닫기 버튼 |
| `EmptyState` | 프로젝트가 0개 일 때 가운데 안내 |
| `Prompt` | 이름 입력용 가벼운 모달 (`window.prompt` 대체 가능) |

## 4. 인터랙션 플로우

### 4.1 프로젝트 생성

1. 사이드바 헤더 `＋` 클릭.
2. 이름 입력 모달 표시 → 사용자가 입력 후 Enter.
3. 폴더 선택 다이얼로그(`dialog:pickFolder`) 표시 → 폴더 확정 또는 취소.
4. 프로젝트 생성과 동시에 디폴트 셸로 **첫 터미널 탭 1개 자동 생성**.
5. 사이드바 활성 프로젝트가 신규 항목으로 전환되고, 탭바·터미널이 그려진다.

### 4.2 탭 추가

1. 탭바 `＋` 클릭.
2. 새 탭의 패널은 프로젝트 폴더에서 디폴트 셸로 즉시 시작.
3. 탭이 활성화되고 자동 포커스.

### 4.3 셸 선택 (프로젝트 디폴트 변경)

1. 사이드바 하단 `shell:` 드롭다운 클릭 → cmd / PowerShell / WSL 목록.
2. WSL이 가용하지 않으면 비활성 상태로 표시 (`WSL (설치되지 않음)`).
3. 선택 시 활성 프로젝트의 `defaultShell` 저장. **기존 탭의 셸은 그대로 두고, 이후 새로 만드는 탭/패널부터 적용**.

### 4.4 탭 분할

1. 탭바 우측 `⊟▢`(좌우) 또는 `⊟▭`(상하) 클릭.
2. 현재 활성 패널이 두 개로 분할되며 새 패널이 디폴트 셸로 시작.
3. 가운데 splitter 드래그로 비율 조정. 비율은 0.1 ~ 0.9 사이로 제한.
4. 한 탭당 분할 깊이는 최대 2 (2×2 = 4패널).

### 4.5 패널 닫기 / 분할 해제

1. 활성 패널 헤더의 `×` 클릭.
2. 형제 패널이 부모 위치를 차지하며 분할이 해제. 단일 패널 남으면 일반 탭 상태로 복귀.
3. 마지막 패널을 닫으면 해당 **탭 전체가 닫힌다**.

## 6. 접근성·키보드 기본 동작

- Tab 키로 사이드바 → 탭바 → 활성 패널 순환 포커스.
- Esc 로 모달 닫기.
- 터미널 포커스 상태에서는 모든 키 입력이 PTY로 전달 (앱 단축키는 v2에서 도입).
- 색 대비 WCAG AA 충족(`--fg` vs `--bg-app`).

## 7. 반응형

- 윈도우 폭 < 720px: 사이드바를 토글형으로 축소(아이콘만). 햄버거 버튼으로 펼침.
- 최소 지원 윈도우 크기: 720 × 480.

## 8. 빈/오류 상태 처리

| 상태 | 표시 |
|---|---|
| 프로젝트 0개 | EmptyState + 새 프로젝트 CTA |
| 프로젝트는 있지만 탭 0개 | 탭바에 `＋` 만 보이고, 메인 영역에 "탭을 추가하세요" 안내 |
| 폴더가 삭제됨 | 셸 시작 실패 후 사용자 홈에서 셸 띄움 + 상단에 경고 배너 |
| PTY 비정상 종료 | 패널 내부 헤더에 종료 메시지 + 재시작 버튼 |

## 9. 향후 디자인 고려 (참고용, 구현 X)

- 명령 팔레트(Ctrl+P).
- 검색 결과 강조(`Ctrl+F`).
- 라이트 테마와 토큰 매핑.
